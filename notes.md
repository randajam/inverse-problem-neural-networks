# Notes

1. При использовании равномерной сетки разбиения параметров 
    $h,\; k,\; \theta$ 
    количество возможных комбинаций параметров экспоненциально растёт с увеличением числа слоёв $N$, 
    что приводит к проклятию размерности и делает генерацию обучающей выборки вычислительно неосуществимой.

    В частности, даже при небольшом числе дискретных значений каждого параметра общее число комбинаций 
    становится недопустимо большим для прямого перебора:

    $N=5$, на каждый параметр по 5 значений $\to$ количество комбинаций $5^{13} = 1.22*10^{9}$

    Возможные стратегии уменьшения размерности пространства параметров:
    - **Latin Hypercube Sampling (LHS)** — равномерное покрытие пространства параметров без экспоненциального роста числа точек;
    - **Иерархическая генерация данных** — постепенное увеличение сложности модели (числа слоёв) и диапазонов параметров;
    - **Случайная (стохастическая) выборка** с последующей фильтрацией физически неинформативных конфигураций;
    - **Активное обучение** — обучение предварительной модели и последующее досэмплирование в областях параметрического пространства с максимальной ошибкой аппроксимации.
